
# Neo Brutalist Design for MantleMandate-SaaS

### 1. **Dashboard for Performance Tracking**
   - **Layout**: Use a **grid-based, rigid layout**. The dashboard should have clear visual separation, with blocks of information that are **boxed in**. These should be presented in **high contrast** colors to ensure that each metric (e.g., **ROI**, **Trading Volume**) stands out.
   - **Typography**: Large, bold **sans-serif** fonts in uppercase for headings like **NAV**, **ROI**, and **Trading Volume**. The text should be clear and functional, avoiding unnecessary decoration.
   - **UI Elements**:
     - **Boxes with visible borders**: Each performance metric (e.g., **ROI**, **Net Asset Value**) should be inside a distinct **box** with **bold borders** to define the content clearly.
     - **Minimal color palette**: Use **black**, **white**, and accent **blue (#0084C7)** for primary actions (e.g., buttons or links). **Red** can be used for error or alert messages.

### 2. **Plain-English Mandate Editor**
   - **Layout**: The interface should feel **utilitarian**—input fields and buttons should be arranged in **rigid rows or columns** without much visual softness. Each field has clearly visible boundaries.
   - **Typography**: Use **monospace fonts** or **bold grotesque fonts** for all text. The font should be easily scannable, with tight letter spacing to create a more **raw feel**.
   - **UI Elements**:
     - **High contrast, boxy input fields**: Use sharp-edged, boxed input fields for mandate creation.
     - **Buttons with hard edges**: Action buttons should be **solid and square** with bold text, using colors like **blue (#0084C7)** for primary actions and **red** for cancellation.
     - **Hover effects**: Hover states should be sharp, with **sudden color swaps**.

### 3. **AI Trading Agent Deployment & Monitoring**
   - **Layout**: Implement **panels** to show the **status** of the AI agent. Each panel should have clearly defined borders and a **high-contrast background** for the agent's name, performance metrics, and status.
   - **Typography**: Use **uppercase**, **monospace**, or **fixed-width fonts** for the code-like feel of agent status.
   - **UI Elements**:
     - **Card-based containers**: Use **hard-edge cards** for each AI agent. Each card can contain metrics like **performance**, **agent status**, and **trading volume**.
     - **Strong borders** around each element: Ensure that each **button**, **text area**, and **status box** stands out with sharp **border outlines**.
     - **Iconography**: Use simple icons that reflect the function, with **bold outlines** and **flat** designs.

### 4. **On-Chain Audit Viewer**
   - **Layout**: Use a **list-based view** for transaction logs with **clear dividers** between each transaction.
   - **Typography**: Large, **bold**, **monospace fonts** for transaction IDs and hashes.
   - **UI Elements**:
     - **Data tables**: Present **transaction logs** and **hashes** in a **table layout**, where each row has clear, visible lines separating each element (Transaction ID, Date, Amount, Status).
     - **Click-to-expand links**: For transaction details, use **underlined links** that turn into a sharp color on hover.

### 5. **Real-Time Alerts**
   - **Layout**: Alerts should appear as **sharp-edged banners** that pop in at the top or bottom of the screen.
   - **Typography**: Use **bold** sans-serif fonts in **uppercase** for alert titles (e.g., **TRADE SUCCESSFUL**, **INSUFFICIENT FUNDS**).
   - **UI Elements**:
     - **Alert Banners**: Display alert banners with **solid colors** (e.g., red for errors, green for success) and clear **black text**.
     - **Hover States**: Use **sharp color transitions** for hover effects on any clickable buttons or alerts.

### 6. **Reports & Exporting**
   - **Layout**: The report section should present data in a **grid format** with **clearly defined lines**.
   - **Typography**: Use **large, bold headers** for report titles and **smaller, lighter text** for descriptions. The font should be **functional**, making the content easy to read at a glance.
   - **UI Elements**:
     - **Export buttons**: The buttons to **download** reports should be simple with **bold edges** and **sharp text**.
     - **No extra decoration**: Reports should be purely functional, with no unnecessary **visual elements** that distract from the core data.

### 7. **Security and Compliance**
   - **Layout**: Security features should have a **structured layout**, with each section of the UI separated by **rigid lines**.
   - **Typography**: Use **monospace** or **bold** fonts for data such as wallet addresses, **multisig setups**, and permissions.
   - **UI Elements**:
     - **Buttons for actions** (e.g., Confirm, Authorize): Use **flat**, **boxy buttons** with bold text.
     - **Confirmation modals**: Ensure confirmation modals for security actions have **clear, minimal text** and **high contrast** colors.

### 8. **Multi-Protocol Integration**
   - **Layout**: Use a **grid layout** to list available protocols, with **bordered panels** separating each one.
   - **Typography**: Use bold, **uppercase** typography to label each protocol.
   - **UI Elements**:
     - **Protocol Panels**: Each protocol can be a **boxy panel** with **flat icons** representing each one (e.g., **Merchant Moe**, **Agni Finance**).
     - **Clear Action Buttons**: Place action buttons like **Connect**, **Execute**, and **Monitor** within the protocol panel.

### 9. **API Integration & Data Ingestion**
   - **Layout**: This section should be **simple and technical**, with **clear box-like containers** for input and output data.
   - **Typography**: Use **monospace** or **fixed-width fonts** to make it feel like an **IDE** or **command-line interface**.
   - **UI Elements**:
     - **Data Panels**: Show **API data** results in distinct, **bordered boxes**.
     - **Error Notifications**: Use **sharp error messages** in red with bold, **monospace fonts**.

### 10. **Risk Engine & Venue Planner**
   - **Layout**: Use **input fields and sliders** arranged in **clear rows** for risk thresholds and venue selections.
   - **Typography**: Use **bold, uppercase** for headers and **monospace** for risk-related text.
   - **UI Elements**:
     - **Risk Threshold Sliders**: Implement **sliders** for setting risk thresholds with **clear labels**.
     - **Action Buttons**: Use **sharp, high-contrast buttons** for setting or confirming changes.

# 20. Payment Methods Section

## **Layout**:
The page should present users with the option to **add, edit, or remove payment methods**. It should have clear sections for **current payment methods**, **billing history**, and **payment settings**.

## **Typography**:
- Use **bold** for section headings such as **"Payment Methods"**, **"Billing History"**, and **"Payment Settings"**.
- Use **clear sans-serif fonts** for form labels like **"Card Number"**, **"Expiration Date"**, and **"Billing Address"**.

## **UI Elements**:
- **Current Payment Method**: Display the active payment method with details like **last 4 digits of card number**, **expiry date**, and **billing address**.
- **Add New Payment Method**: Provide a prominent button to **add a new payment method**.
- **Edit/Remove Options**: Enable users to **edit or remove** their existing payment methods with **clear call-to-action buttons**.
- **Payment History**: Show a **list of past transactions** and their statuses.
- **Payment Method Icons**: Use icons for popular payment methods such as **Visa**, **Mastercard**, **PayPal**, etc.

## **Payment Method Elements**:
- **Credit/Debit Card**: Allow users to enter their card details (card number, expiration date, CVV).
- **Bank Account**: If applicable, provide an option for users to **link a bank account**.
- **PayPal**: Include PayPal as an alternative payment option for users.
- **Subscription Plans**: Allow users to change their subscription plans (e.g., from **Basic** to **Pro**) directly on the page.

## **UI Interactions**:
- **Input Fields**: Provide **secure input fields** for payment information, ensuring they are clearly defined and include appropriate error messages (e.g., for invalid card numbers or expiration dates).
- **CTA Buttons**: Clear **action buttons** like **"Save Payment Method"** and **"Cancel"**.
- **Error Notifications**: Display any errors (e.g., "Card declined", "Invalid expiration date") in a clear **red notification** in **monospace font**.
- **Confirmation Modals**: After adding or editing a payment method, show a **confirmation modal** to ensure the changes were saved successfully.

### **General Design Guidelines for Neo Brutalism in MantleMandate-SaaS**
- **Grid-Based Layout**: All elements should follow a strict, visible grid, with clear boundaries around each section.
- **High Contrast**: Use **dark backgrounds** with **white or bright text**. Color contrast should be bold (e.g., black, white, blue #0084C7).
- **Bold Typography**: Use **uppercase**, **monospace** fonts for a more technical, utilitarian feel.
- **Functional over Decorative**: Avoid unnecessary animations or flourishes. Focus purely on function and clarity.
- **Responsive Design**: Ensure the design adapts smoothly on mobile, using stacked panels and a flexible grid system.
